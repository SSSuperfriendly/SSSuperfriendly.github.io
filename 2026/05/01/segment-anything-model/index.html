SAM：从点一下抠图到概念分割

我第一次系统看 SAM 的时候，最容易被它的演示效果带偏。点一下，出来一个 mask；画个框，边界也很干净。这个效果很直观，传播起来也方便，但如果只停在“抠图工具”这一层，反而会低估 SAM 在计算机视觉里的位置。SAM 真正有意思的地方不是它又做了一个更大的分割模型，而是它把分割任务改写成了一个可以被 prompt 调用的接口。

这个变化和 NLP 里的 prompt 范式有相似之处，但不能简单类比成“图像版 GPT”。GPT 的核心动作是根据上下文自回归地产生下一个 token，SAM 的核心动作则是在给定图像和提示之后预测像素级 mask。它不需要逐像素生成，也不是一个开放式语言模型。更准确地说，SAM 是一个可提示的视觉分割基础模型：图像先被编码成可复用的视觉表示，点、框、粗 mask 等提示再被编码成条件，最后由一个轻量 mask decoder 把图像表示和提示条件合成目标区域。

这也是理解 SAM 技术史时最应该抓住的线索。传统分割模型通常把任务和训练数据绑定得很死。语义分割要在固定类别集合上预测每个像素的类别，实例分割要先找到对象，再给出每个实例的 mask。模型在训练时见过什么类别、什么数据域，往往决定了它在测试时能做什么。SAM 则把“要分割什么”推迟到推理阶段，由 prompt 指定。它试图学习的不是某个固定类别表，而是一种更通用的对象边界和区域选择能力。

SAM 论文里最重要的定义可以写成下面这个形式：

fθ(I,p)→{(M^i,q^i)}i=1K.f_\theta(I, p) \rightarrow \{(\hat{M}_i, \hat{q}_i)\}_{i=1}^{K}. fθ​(I,p)→{(M^i​,q^​i​)}i=1K​.

这里 III 是图像，ppp 是提示，M^i∈[0,1]H×W\hat{M}_i \in [0,1]^{H \times W}M^i​∈[0,1]H×W 是第 iii 个候选 mask，q^i\hat{q}_iq^​i​ 是模型估计的 mask 质量，通常对应预测 IoU。这个接口看起来简单，但它把分割问题从“对固定类别做像素分类”改成了“对一个由提示指定的对象返回有效 mask”。后者才是 SAM 后续可以接入交互式标注、自动 mask 生成、视频跟踪和概念分割的原因。

SAM 项目本身也不是单纯的模型论文。它包含三个互相支撑的部分：一个 promptable segmentation task，一个 SAM 模型，以及一个用模型参与标注的数据引擎。很多介绍会把注意力放在 ViT、prompt encoder 和 mask decoder 上，这当然重要，但如果没有 SA-1B 数据引擎，SAM 很难成为现在这个意义上的 SAM。分割 mask 不像文本那样天然存在于互联网上，人工描边又很贵，所以 Meta 实际上构造了一个模型和数据互相放大的闭环。

1. 分割任务为什么需要重新定义

在 SAM 之前，分割技术已经发展了很久。早期方法依赖阈值、边缘、graph cut、active contour、GrabCut 这类图像处理或优化算法。它们在可控场景下很有用，但语义能力有限。深度学习之后，FCN、U-Net、DeepLab、Mask R-CNN、Mask2Former 等模型让分割变成了监督学习问题，效果明显提升，代价是任务边界也更清楚地被数据集固定下来。

语义分割通常学习一个映射：

I→{yuv∈C}u,v.I \rightarrow \{y_{uv} \in \mathcal{C}\}_{u,v}. I→{yuv​∈C}u,v​.

也就是说，每个像素都要落到类别集合 C\mathcal{C}C 中。实例分割则更像：

I→{(bj,cj,Mj)}j=1N.I \rightarrow \{(b_j, c_j, M_j)\}_{j=1}^{N}. I→{(bj​,cj​,Mj​)}j=1N​.

模型要输出边界框 bjb_jbj​、类别 cjc_jcj​ 和实例 mask MjM_jMj​。这两种形式都很自然，也都很有效，但它们不太适合作为“通用分割接口”。如果测试时出现了训练类别表之外的对象，或者用户只想分割某个没有类别名的局部结构，传统接口就会显得笨重。

SAM 的做法是把任务写成：

(I,p)→valid mask.(I, p) \rightarrow \text{valid mask}. (I,p)→valid mask.

这个 ppp 可以是前景点、背景点、框，也可以是上一轮产生的粗 mask。SAM 1 论文中确实探索过自由文本提示，但公开模型最常用、最成熟的接口仍然是空间提示。这个边界需要说清楚，SAM 1 并不是一个成熟的开放词汇文本分割系统。它更像一个通用的几何分割器，用户用点、框或 mask 告诉它关注哪里，它再沿着图像中的对象结构返回一个合理区域。

“valid mask” 这个定义很关键。一个点本来就可能有多种解释。点在自行车轮胎上，合理输出可以是轮胎、车轮，也可以是整辆自行车。点在一个人的衣服上，合理输出可以是衣服，也可以是整个人。如果强迫模型只给一个答案，训练时就会把多个合理目标揉成一个平均结果，边界很容易变得含糊。SAM 因此保留多个候选 mask，并为每个候选预测质量分数。

这个设计可以写成：

fθ(I,p)={(M^1,q^1),(M^2,q^2),(M^3,q^3)}.f_\theta(I,p)=\{(\hat{M}_1,\hat{q}_1),(\hat{M}_2,\hat{q}_2),(\hat{M}_3,\hat{q}_3)\}. fθ​(I,p)={(M^1​,q^​1​),(M^2​,q^​2​),(M^3​,q^​3​)}.

训练时，模型不会要求每个候选都贴近同一个目标，而是对多个候选取最小损失：

Lmask=min⁡i[λfLfocal(M^i,M⋆)+λdLdice(M^i,M⋆)].\mathcal{L}_{mask} = \min_i \left[ \lambda_f \mathcal{L}_{focal}(\hat{M}_i,M^\star) + \lambda_d \mathcal{L}_{dice}(\hat{M}_i,M^\star) \right]. Lmask​=imin​[λf​Lfocal​(M^i​,M⋆)+λd​Ldice​(M^i​,M⋆)].

论文中 focal loss 和 dice loss 的权重大致是 20:1。focal loss 主要处理前景和背景像素数量极不平衡的问题，dice loss 则直接关心区域重叠。Dice loss 的常见写法是：

Ldice(M^,M)=1−2∑u,vM^uvMuv+ϵ∑u,vM^uv+∑u,vMuv+ϵ.\mathcal{L}_{dice}(\hat{M},M) = 1- \frac{2\sum_{u,v}\hat{M}_{uv}M_{uv}+\epsilon} {\sum_{u,v}\hat{M}_{uv}+\sum_{u,v}M_{uv}+\epsilon}. Ldice​(M^,M)=1−∑u,v​M^uv​+∑u,v​Muv​+ϵ2∑u,v​M^uv​Muv​+ϵ​.

SAM 还会用 IoU head 学习估计候选 mask 的质量：

Liou=(q^i−IoU(1[M^i>τ],M⋆))2.\mathcal{L}_{iou} = \left( \hat{q}_i - \mathrm{IoU}(\mathbb{1}[\hat{M}_i>\tau],M^\star) \right)^2. Liou​=(q^​i​−IoU(1[M^i​>τ],M⋆))2.

所以完整目标可以理解为：

L=min⁡iLseg(i)+λiouLiou(i).\mathcal{L} = \min_i \mathcal{L}_{seg}^{(i)} + \lambda_{iou}\mathcal{L}_{iou}^{(i)}. L=imin​Lseg(i)​+λiou​Liou(i)​.

这组公式背后的思想并不复杂。SAM 承认 prompt 可能有歧义，所以让不同 mask token 学会覆盖不同解释，再用预测 IoU 帮用户或系统选择更可靠的结果。

2. Image Encoder：昂贵但可缓存的视觉表示

SAM 的 image encoder 是一个 ViT。输入图像通常被缩放并 padding 到 1024×10241024 \times 10241024×1024，patch size 为 16，于是空间 token 网格为：

102416×102416=64×64.\frac{1024}{16}\times\frac{1024}{16}=64\times64. 161024​×161024​=64×64.

Patch embedding 可以看成一个 kernel size 和 stride 都为 16 的卷积：

I∈R3×1024×1024→X0∈R64×64×C.I\in\mathbb{R}^{3\times1024\times1024} \rightarrow X_0\in\mathbb{R}^{64\times64\times C}. I∈R3×1024×1024→X0​∈R64×64×C.

如果展平成序列，就是 409640964096 个视觉 token：

X0∈R4096×C.X_0 \in \mathbb{R}^{4096\times C}. X0​∈R4096×C.

后面的 Transformer block 仍然基于标准 attention：

Attn(Q,K,V)=softmax(QK⊤d)V.\mathrm{Attn}(Q,K,V) = \mathrm{softmax}\left(\frac{QK^\top}{\sqrt{d}}\right)V. Attn(Q,K,V)=softmax(d​QK⊤​)V.

问题在于图像的 token 数比普通文本句子大得多。如果在 409640964096 个 token 上直接做全局 attention，注意力矩阵规模是：

40962≈1.68×107.4096^2 \approx 1.68\times10^7. 40962≈1.68×107.

这对显存和计算都不便宜。SAM 的 ViT 因此混合使用窗口注意力和少量全局注意力。窗口注意力只在局部窗口内计算 attention，如果窗口大小是 w×ww\times ww×w，复杂度从全局的

O((HW)2)O((HW)^2) O((HW)2)

降到近似

O(HW⋅w2).O(HW\cdot w^2). O(HW⋅w2).

少量全局 attention 层再负责补充长距离信息。这个设计不是新的理论口号，而是高分辨率视觉 Transformer 必须面对的工程折中。大多数层处理局部纹理、边界和形状，少数层传递全局上下文。

SAM 还使用二维相对位置编码。对图像 token 来说，attention logits 不只来自 qi⊤kjq_i^\top k_jqi⊤​kj​，还应该知道两个 patch 在二维平面上的相对位置。可以写成：

Aij=qi⊤kjd+bΔx(i,j)+bΔy(i,j).A_{ij} = \frac{q_i^\top k_j}{\sqrt{d}} + b_{\Delta x(i,j)} + b_{\Delta y(i,j)}. Aij​=d​qi⊤​kj​​+bΔx(i,j)​+bΔy(i,j)​.

源码中的 decomposed relative position 把二维偏置拆成横向和纵向两个一维表。这样参数更省，也保留了必要的空间几何关系。

image encoder 最后会通过 neck 把 ViT 输出投影到 mask decoder 需要的固定通道数，常见形状是：

R64×64×C→R256×64×64.\mathbb{R}^{64\times64\times C} \rightarrow \mathbb{R}^{256\times64\times64}. R64×64×C→R256×64×64.

这一步没有神秘之处，本质上是表示适配。ViT 主干负责产生强视觉表示，decoder 只需要一个尺寸稳定、通道数合适的特征图。

从系统设计看，image encoder 的重计算是可以接受的，因为它只需要对一张图算一次。用户在同一张图上继续点选、加框、修正 mask 时，后续只需要重新跑 prompt encoder 和 mask decoder。这是 SAM 能做成交互式工具的关键。

3. Prompt Encoder：把提示变成模型能用的条件

Prompt encoder 解决的是一个很具体的问题：点、框和 mask 不是同一种数据，但 decoder 需要在同一个空间里使用它们。SAM 因此把 prompt 分成稀疏提示和稠密提示。点和框数量少，适合编码成 token；mask 本身是空间图，适合编码成和 image embedding 对齐的特征图。

一个点包含坐标和标签。坐标 c=(x,y)c=(x,y)c=(x,y) 先归一化到 [0,1]2[0,1]^2[0,1]2，再经过随机 Fourier 位置编码：

γ(c)=[sin⁡(2πBc),cos⁡(2πBc)],\gamma(c) = \left[ \sin(2\pi Bc), \cos(2\pi Bc) \right], γ(c)=[sin(2πBc),cos(2πBc)],

其中 B∈Rm×2B\in \mathbb{R}^{m\times2}B∈Rm×2 是随机高斯矩阵。模型还会根据点类型加入可学习 embedding：

epoint=γ(c)+tℓ.e_{\text{point}} = \gamma(c)+t_{\ell}. epoint​=γ(c)+tℓ​.

ℓ\ellℓ 表示前景点、背景点或 padding 点。这样模型不仅知道点在哪里，也知道这个点是在鼓励前景还是排除背景。

框 prompt 可以看成两个角点。给定

b=(x1,y1,x2,y2),b=(x_1,y_1,x_2,y_2), b=(x1​,y1​,x2​,y2​),

SAM 会把它拆成左上和右下两个坐标，并分别加入不同的 corner embedding：

ebox=[γ(x1,y1)+tcorner1,γ(x2,y2)+tcorner2].e_{\text{box}} = \left[ \gamma(x_1,y_1)+t_{corner1}, \gamma(x_2,y_2)+t_{corner2} \right]. ebox​=[γ(x1​,y1​)+tcorner1​,γ(x2​,y2​)+tcorner2​].

这说明框在模型内部并不是一个特殊的矩形结构，而是两个带类型的空间 token。这样的处理方式很朴素，也很方便和点 prompt 共用同一套 decoder。

mask prompt 则不同。给定上一轮 mask logits 或一个粗 mask M0M_0M0​，SAM 用小卷积网络把它下采样到 image embedding 的空间尺度：

Ed=ϕmask(M0)∈R256×64×64.E_d=\phi_{mask}(M_0)\in \mathbb{R}^{256\times64\times64}. Ed​=ϕmask​(M0​)∈R256×64×64.

然后直接加到图像特征上：

E~I=EI+Ed.\tilde{E}_I=E_I+E_d. E~I​=EI​+Ed​.

这里可以看到 sparse prompt 和 dense prompt 的区别。点和框是作为 token 进入 decoder，mask 则是作为空间先验直接调制图像特征。前者回答“目标在哪里”，后者已经给出一张粗略的目标区域。

4. Mask Decoder：SAM 的核心算法

SAM 的 mask decoder 接收三类信息：图像特征、提示特征和一组可学习的输出 token。它的输入可以抽象为：

T0=[tiou,tmask(0),tmask(1),...,tmask(K),Es],T_0=[t_{iou},t_{mask}^{(0)},t_{mask}^{(1)},...,t_{mask}^{(K)},E_s], T0​=[tiou​,tmask(0)​,tmask(1)​,...,tmask(K)​,Es​],

以及

X0=EI+Ed.X_0=E_I+E_d. X0​=EI​+Ed​.

这里的 tiout_{iou}tiou​ 和 tmask(i)t_{mask}^{(i)}tmask(i)​ 都是可学习 token。它们和 DETR 里的 object queries 有相同的思想：先放几个空槽位，再让模型通过 attention 从图像里取回对应答案。mask token 负责形成候选分割结果，IoU token 负责估计这些结果的质量。

SAM 的 decoder 不是 GPT 式自回归 decoder。它不会先生成第一个像素，再生成第二个像素，也不会按时间顺序逐步展开一个序列。它更像一个并行 query decoder。Two-Way Transformer 的一个 block 可以写成四个操作。

token 首先在内部做 self-attention，让 IoU token、mask token 和 prompt token 彼此交换信息：

T1=LN(T0+Attn(T0+PT,T0+PT,T0)).T_1 = \mathrm{LN}\left(T_0+\mathrm{Attn}(T_0+P_T,T_0+P_T,T_0)\right). T1​=LN(T0​+Attn(T0​+PT​,T0​+PT​,T0​)).

随后 token 去查询图像特征：

T2=LN(T1+Attn(T1+PT,X0+PX,X0)).T_2 = \mathrm{LN}\left(T_1+\mathrm{Attn}(T_1+P_T,X_0+P_X,X_0)\right). T2​=LN(T1​+Attn(T1​+PT​,X0​+PX​,X0​)).

再经过一个 MLP：

T3=LN(T2+MLP(T2)).T_3 = \mathrm{LN}\left(T_2+\mathrm{MLP}(T_2)\right). T3​=LN(T2​+MLP(T2​)).

最后图像特征反过来查询 token：

X1=LN(X0+Attn(X0+PX,T3+PT,T3)).X_1 = \mathrm{LN}\left(X_0+\mathrm{Attn}(X_0+P_X,T_3+P_T,T_3)\right). X1​=LN(X0​+Attn(X0​+PX​,T3​+PT​,T3​)).

最后一步是 Two-Way 的关键。标准 Transformer decoder 通常只是让 query 去看 encoder 输出，SAM 还让图像特征被 prompt token 反向调制。原因很直接：最后的 mask 是由更新后的 image features 和 mask tokens 一起算出来的。如果图像特征完全不受 prompt 影响，同一张图上点猫和点狗，后续空间特征会缺少足够的目标条件。

Two-Way Transformer 之后，每个 mask token 经过一个小 MLP，生成一个动态线性分类器：

wi=MLPi(tmask(i)).w_i = \mathrm{MLP}_i(t_{mask}^{(i)}). wi​=MLPi​(tmask(i)​).

图像特征经过转置卷积上采样为：

F↑∈RC′×H′×W′.F^\uparrow \in \mathbb{R}^{C'\times H'\times W'}. F↑∈RC′×H′×W′.

每个像素位置的 mask logit 由点积得到：

M^i(u,v)=wi⊤Fu,v↑.\hat{M}_i(u,v) = w_i^\top F^\uparrow_{u,v}. M^i​(u,v)=wi⊤​Fu,v↑​.

这个动态 head 很简洁。mask token 不是直接吐出一张 mask，而是生成一组当前候选 mask 的像素分类权重。图像特征给出每个位置的局部表示，mask token 决定当前要用什么判别方向去切分这些像素。

IoU token 通过另一个 MLP 产生质量估计：

q^=MLPiou(tiou).\hat{q}=\mathrm{MLP}_{iou}(t_{iou}). q^​=MLPiou​(tiou​).

推理时，系统可以按 q^\hat{q}q^​ 对候选 mask 排序，也可以在自动 mask 生成中用它过滤低质量结果。

5. 推理和训练并不是两件孤立的事

交互式推理时，SAM 先对图像做预处理，将长边缩放并 padding 到固定尺寸，再通过 image encoder 得到 EIE_IEI​。这一步只算一次。用户给出点、框或 mask 后，prompt encoder 产生 EsE_sEs​ 和 EdE_dEd​，mask decoder 输出多个候选 mask 和对应 IoU 分数。低分辨率 mask 会被上采样回原图尺寸，前端再根据质量分数展示最合适的结果。

自动 mask 生成只是把提示采样程序化。系统会在图像上布置网格点，把每个点当成前景点调用 SAM，收集多个候选 mask，再根据 predicted IoU、stability score 和 NMS 去掉不稳定或重复的结果。必要时还会对图像 crop 重复这个过程，以便找到小目标。这里的 stability score 可以理解为一个鲁棒性检查：如果对同一张概率图用两个相近阈值得到的二值 mask 差异很大，这个预测就不稳定。

训练阶段也在模拟这种交互。对一个真值 mask，模型会先接收一个初始 prompt，可以是 mask 内的前景点，也可以是带噪声的框。模型预测后，训练过程会在错误区域继续采样修正点。假如漏掉了一块前景，就在 false negative 区域采样一个前景点；假如多分出一块背景，就在 false positive 区域采样一个背景点。上一轮的 mask logits 也会作为新的 mask prompt 输入模型。论文中的训练细节使用了多轮交互模拟，让模型学会从少量提示逐步修正结果。

这套训练方式和数据引擎是配套的。模型越能用少量点击得到好 mask，人工标注越快；标注越快，越能生成更多训练 mask；更多 mask 又反过来提升模型。这正是 SAM 项目和普通分割模型不一样的地方。

6. SA-1B：模型能力的一半来自数据引擎

SAM 的泛化能力不能只归因于 ViT-H 或 mask decoder。分割 mask 是稀缺监督信号，不能像网页文本那样直接爬取。SA-1B 数据集包含约 1100 万张图像和 11 亿个 mask，它不是凭空出现的，而是由数据引擎逐步构造出来的。

早期阶段，标注员使用 SAM 辅助工具进行交互式标注。模型提供初始 mask，标注员用点和框修正，得到高质量人工 mask。随着模型变强，流程进入半自动阶段，模型先生成一部分高置信 mask，标注员主要补充遗漏对象。最后，全自动阶段在图像网格上大规模运行 SAM，再用 predicted IoU、stability score、NMS 和多 crop 策略筛掉明显不可靠的 mask。

这个过程的本质是一个数据飞轮：

better model→faster annotation→more masks→better model.\text{better model} \rightarrow \text{faster annotation} \rightarrow \text{more masks} \rightarrow \text{better model}. better model→faster annotation→more masks→better model.

对视觉基础模型来说，这个飞轮比单个模块更值得重视。没有 SA-1B，SAM 可能只是一个设计不错的交互式分割网络；有了 SA-1B，它才真正具备基础模型意义上的零样本迁移能力。

7. SAM 2：图像分割进入时间维度

SAM 2 并不是逐帧运行 SAM 1。视频分割的问题不只是“每一帧都分得准”，还要在遮挡、运动、形变和目标消失后保持对象身份。静态图像中，一个点 prompt 只需要指定当前对象；视频中，同一个对象要沿时间轴持续存在。

SAM 2 把 promptable segmentation 扩展成图像和视频统一的 Promptable Visual Segmentation。给定一段视频 VVV 和某一帧上的提示 pt0p_{t_0}pt0​​，模型要输出跨帧 mask：

(V,pt0)→{Mt}t=1T.(V, p_{t_0}) \rightarrow \{M_t\}_{t=1}^{T}. (V,pt0​​)→{Mt​}t=1T​.

它的核心是 streaming memory。对第 ttt 帧，模型先用 image encoder 得到当前帧特征：

Xt=Eimg(It).X_t = E_{img}(I_t). Xt​=Eimg​(It​).

随后 memory attention 用历史 memory bank Bt−1B_{t-1}Bt−1​ 调制当前帧特征：

X~t=MemAttn(Xt,Bt−1).\tilde{X}_t = \mathrm{MemAttn}(X_t,B_{t-1}). X~t​=MemAttn(Xt​,Bt−1​).

mask decoder 再根据当前帧的条件特征输出 mask：

M^t=D(X~t,pt).\hat{M}_t = D(\tilde{X}_t,p_t). M^t​=D(X~t​,pt​).

当前预测会被 memory encoder 写回 memory bank：

Bt=FIFO(Bt−1∪MemEnc(Xt,M^t)).B_t = \mathrm{FIFO}\left(B_{t-1}\cup \mathrm{MemEnc}(X_t,\hat{M}_t)\right). Bt​=FIFO(Bt−1​∪MemEnc(Xt​,M^t​)).

SAM 2 的 memory bank 既保存空间 memory feature，也保存 object pointer 这类轻量对象向量。前者记录像素级外观，后者提供较高层的对象身份线索。它还需要处理目标是否被遮挡、是否暂时消失这类视频中特有的问题。换句话说，SAM 2 的关键不是“支持视频”这句表层描述，而是把 SAM 1 的空间查询改造成了带记忆的时空查询。

8. SAM 3：从位置提示走向概念提示

SAM 1 和 SAM 2 的主要限制在于语义。你点一个对象，它可以给出 mask；你说“找出所有黄色校车”，它本身并不能完成开放词汇概念搜索。SAM 3 的任务因此转向 Promptable Concept Segmentation：

(I or V, c)→{(Mj,idj)}j=1Nc.(I \text{ or } V,\ c) \rightarrow \{(M_j,\mathrm{id}_j)\}_{j=1}^{N_c}. (I or V, c)→{(Mj​,idj​)}j=1Nc​​.

这里 ccc 是 concept prompt，可以是短名词短语，也可以是图像 exemplar，或二者组合。需要注意的是，SAM 3 论文中的文本提示主要是简单 noun phrase，例如 “yellow school bus” 或 “striped cat”。它不是面向长指代表达和复杂语言推理的通用分割器。

SAM 3 的结构可以理解为检测器和跟踪器解耦。检测器在单帧里根据 concept prompt 找到所有匹配实例，跟踪器继承 SAM 2 的 memory-based 思路，在视频中维护身份一致性。二者共享视觉语言表示，但优化目标不同。检测器关心当前帧里哪些区域属于目标概念，跟踪器关心这些区域在时间上是否对应同一个对象。

检测器部分继承 DETR 式 object query。图像和文本或样例 prompt 先被编码成视觉语言特征，fusion encoder 让图像特征和概念提示交互：

Z=Fusion(X,Pc).Z=\mathrm{Fusion}(X,P_c). Z=Fusion(X,Pc​).

DETR decoder 中的一组 object queries 再查询融合后的特征：

oj=Decoder(qj,Z).o_j = \mathrm{Decoder}(q_j,Z). oj​=Decoder(qj​,Z).

每个 query 预测框、mask 和局部匹配得分：

sjloc=P(yj=1∣oj,Z,c).s_j^{loc}=P(y_j=1\mid o_j,Z,c). sjloc​=P(yj​=1∣oj​,Z,c).

开放词汇场景里，一个常见问题是 hallucination。图中没有目标概念时，query 仍然可能找一个局部上最像的对象。SAM 3 用 presence head 把全局识别和局部定位拆开。Presence head 先回答图像中是否存在这个概念：

spres=P(c exists in image∣Z,Pc).s^{pres}=P(c\ \text{exists in image}\mid Z,P_c). spres=P(c exists in image∣Z,Pc​).

对象 query 再负责定位条件下的局部匹配：

sjloc=P(query j localizes concept∣c present).s_j^{loc}=P(\text{query }j\text{ localizes concept}\mid c\text{ present}). sjloc​=P(query j localizes concept∣c present).

最终分数采用乘法门控：

sj=spres⋅sjloc.s_j=s^{pres}\cdot s_j^{loc}. sj​=spres⋅sjloc​.

这个设计很朴素，但很重要。全局识别需要看整张图，局部定位需要聚焦候选区域。如果把这两件事都压到每个 query 上，模型容易在困难负样本上误检。Presence head 相当于先判断“有没有”，再让 query 去解决“在哪里”。

SAM 3 还引入 classification-gated F1 来评估开放词汇分割。定位质量用正样本上的 micro F1 表示：

pmF1τ=2TPτ2TPτ+FPτ+FNτ.\mathrm{pmF1}^{\tau} = \frac{2TP_{\tau}} {2TP_{\tau}+FP_{\tau}+FN_{\tau}}. pmF1τ=2TPτ​+FPτ​+FNτ​2TPτ​​.

多个 IoU 阈值平均后得到：

pmF1=110∑τ∈{0.50,0.55,…,0.95}pmF1τ.\mathrm{pmF1} = \frac{1}{10} \sum_{\tau\in\{0.50,0.55,\ldots,0.95\}} \mathrm{pmF1}^{\tau}. pmF1=101​τ∈{0.50,0.55,…,0.95}∑​pmF1τ.

图像级是否存在该概念，则用 Matthews correlation coefficient：

IL_MCC=TP⋅TN−FP⋅FN(TP+FP)(TP+FN)(TN+FP)(TN+FN).\mathrm{IL\_MCC} = \frac{ TP\cdot TN-FP\cdot FN }{ \sqrt{ (TP+FP)(TP+FN)(TN+FP)(TN+FN) } }. IL_MCC=(TP+FP)(TP+FN)(TN+FP)(TN+FN)​TP⋅TN−FP⋅FN​.

主指标写成：

cgF1=100⋅pmF1⋅IL_MCC.\mathrm{cgF1}=100\cdot \mathrm{pmF1}\cdot \mathrm{IL\_MCC}. cgF1=100⋅pmF1⋅IL_MCC.

这个乘法结构反映了开放词汇分割的真实要求。模型既要知道概念是否存在，也要在存在时分割得准。只会画边界但经常误报，或者知道图中有目标但 mask 很差，都不够。

9. 回到主线：SAM 到底改变了什么

把 SAM 1、SAM 2 和 SAM 3 放在一起看，主线其实很清楚。SAM 1 解决的是给定空间提示时如何快速返回一个有效 mask。它的关键是可缓存 image embedding、prompt encoder、Two-Way mask decoder、多候选消歧，以及 SA-1B 数据引擎。SAM 2 把这套接口推到视频里，核心变量从静态 image embedding 变成了 memory-conditioned frame embedding：

X~t=MemAttn(Eimg(It),Bt−1).\tilde{X}_t=\mathrm{MemAttn}(E_{img}(I_t),B_{t-1}). X~t​=MemAttn(Eimg​(It​),Bt−1​).

SAM 3 再把提示从几何位置推进到开放词汇概念，关键机制是 presence head：

sj=spres⋅sjloc.s_j=s^{pres}\cdot s_j^{loc}. sj​=spres⋅sjloc​.

这条技术线可以概括为从“点哪里分哪里”，到“点一帧跟全视频”，再到“说一个概念，找出所有实例”。它不是简单地堆大模型，而是在不断改写分割任务的接口。

当然，SAM 系列也不是万能视觉大脑。SAM 1 和 SAM 2 的空间 prompt 能力不等于语义理解能力。它们可以沿着图像边界分割对象，但并不自动知道对象类别、属性和关系。SAM 3 把概念 prompt 放进主线后，开放词汇能力明显增强，但它仍然主要处理名词短语和实例层面的概念搜索。透明物体、反光物体、极细结构、医学影像、遥感小目标、严重遮挡和低清视频仍然会带来失败。

所以我更愿意把 SAM 看成一个强大的 segmentation primitive，而不是完整视觉系统。它可以和检测器、VLM、跟踪器、3D 重建、机器人规划和标注工具组合。它回答的问题很明确：在给定图像和条件后，哪些像素属于目标。真正复杂的应用，往往要把这个 primitive 放进更大的系统里。

如果只用一句话总结，SAM 系列把视觉分割从固定类别的监督任务，推进成了可提示、可组合、可作为系统组件调用的基础模型接口。它的工程壁垒来自任务定义、Transformer 架构、实时交互约束和数据引擎的共同设计，而不是某一个单独模块。
